# 更新日志

## [v4.17.3] - 2026-03-11

### 修复

- **修复 `/算卦设置` 被识别为 `/算卦 设置` 的问题**
  - 在 `divine_keyword` 方法中排除 "算卦设置"、"算卦帮助" 等指令
- **修复 Provider 列表显示类名而不是 ID 的问题**
  - 使用 `p.meta().id` 获取正确的 Provider ID
  - 同时修复 `settings` 和 `help_info` 方法

## [v4.17.2] - 2026-03-11

### 修复

- **修复命令装饰器格式问题**
  - 修复 `@filter.command` 与前一行代码在同一行的问题
  - 此问题导致新命令无法正确注册

## [v4.17.1] - 2026-03-11

### 修复

- **移除 `/算卦` 命令，避免与 `/算卦设置` 冲突**
  - 主命令改为 `/算一卦`

## [v4.17.0] - 2026-03-11

### 新增

- **新增 `/算卦设置` 指令**
  - 支持查看当前配置和可用选项
  - 支持动态切换 LLM 提供商：`/算卦设置 provider <ID>`
  - 支持动态切换人格：`/算卦设置 persona <名称>`
  - 支持重置配置：`/算卦设置 重置`
  - 设置立即生效，重启后恢复为配置文件中的值

- **帮助信息增强**
  - 帮助信息中显示可用的 LLM 提供商列表
  - 显示每个提供商对应的模型名称

## [v4.16.0] - 2026-03-10

### 新增

- **支持指定 LLM 提供商和人格**
  - 新增 `ai_divine_provider_id` 配置项，支持指定 AI 解卦使用的 LLM 提供商 ID
  - 新增 `ai_divine_persona_name` 配置项，支持指定 AI 解卦使用的人格名称
  - 新增 `ai_divine_default_prompt` 配置项，支持自定义默认提示词

### 改进

- 优化 `_get_ai_interpretation` 方法的 Provider 和人格获取逻辑
- 完善错误处理，当指定的 Provider 或人格不存在时自动回退到默认配置
- 更新帮助信息，添加新增配置项说明

### 技术细节

- Provider 获取优先级：插件配置 → 当前会话配置
- 人格获取优先级：插件配置 → 当前会话配置 → 默认提示词
- 使用 `provider_manager.get_provider_by_id()` 获取指定 Provider
- 从 `persona_manager.personas_v3` 列表中查找指定人格
